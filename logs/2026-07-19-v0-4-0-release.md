# 2026-07-19 — v0.4.0 feature release

## Intent

- Publish the accumulated banner, roster, weapon-art, Standard Archive, weapon-selector, and Character Archive work documented in the 2026-07-18 and 2026-07-19 feature logs.
- Designate Ghostty as WishSim's primary supported terminal emulator. Kitty remains supported through the shared graphics protocol and ANSI remains the portable fallback.

## Version and scope

- Release version: `0.4.0` / tag `v0.4.0`.
- Source baseline before release work: `0c6b30f`, the publication record following v0.3.1.
- Release changes are initially in the working tree and must not be described as committed or published until the corresponding steps complete.
- Version metadata changes: `Cargo.toml`, `Cargo.lock`, and `CHANGELOG.md`.
- User-facing terminal-support notes updated in `README.md` without removing Kitty or ANSI behavior.

## Publication plan

1. Run locked formatting, tests, strict Clippy, release build, and diff validation.
2. Commit the complete feature release and push `main`.
3. Create and push annotated tag `v0.4.0`.
4. Wait for the GitHub release workflow and verify all four platform assets.
5. Update both macOS URLs and SHA-256 values in `TMAH1602/homebrew-wishsim`.
6. Run Ruby syntax, Homebrew style/audit/test checks, push the tap, upgrade the installed formula, and verify `wishsim --version`.
