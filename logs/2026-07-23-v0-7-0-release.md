# 2026-07-23 — v0.7.0 feature release

## Intent

- Publish the accumulated Battle Test presentation/mechanics expansion, complete character face portraits, new limited Anemo release, Standard Fate-path selector, standard signature weapons, and six-card Character Management grid.
- Preserve `v0.2.0` as the design-language baseline and Ghostty as the primary supported terminal, with Kitty protocol compatibility and ANSI fallback retained.

## Version and scope

- Release version: `0.7.0` / tag `v0.7.0`.
- Published baseline before release work: v0.6.1 documentation tip `9080f08`, release commit `b75151c`, and tag `v0.6.1`.
- Version metadata changes: `Cargo.toml`, `Cargo.lock`, and `CHANGELOG.md`.
- Detailed behavior and rollback guidance remain in the dated feature logs from `2026-07-23-battle-test-overhaul.md` through `2026-07-23-six-character-roster-grid.md`.
- Release work begins in the working tree and must not be described as committed or published until each publication step succeeds.

## Included feature groups

- Large, visually responsive Battle Test UI with encounter selection, scaled enemy/active-character art, action effects, status indicators, shields, per-character BP, and on-demand battle history.
- Ruin Court, Somnial Frostwyrm, and Mad Goliath encounters with new transparent enemy artwork and distinct tactics.
- BP-only character abilities, expanded support roles, elemental statuses/immunities, weapon-scaled damage, Geo barrier weaknesses, Goliath stun/regeneration lock, and delayed Shardling revival.
- Complete front-facing character portrait collection and a large six-card Character Management quick-selector synchronized across ANSI and Ghostty/Kitty.
- Klara, Jade Tempest; Taisia; Gale's Last Harvest; Klara rename migration; and corrected Klara scythe/arm artwork.
- Five-star standard-character signatures on the weapon path plus a scrollable Standard character Fate-path selector matching the weapon selector.
