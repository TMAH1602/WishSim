## Planned release sequence

1. Set `Cargo.toml` and `Cargo.lock` to `0.3.0`.
2. Move the accumulated feature summary into the `0.3.0 - 2026-07-18` changelog section.
3. Run formatting, tests, Clippy, diff checks, and a release build.
4. Commit all intended source, assets, guides, and dated logs.
5. Create annotated tag `v0.3.0` and push `main` and the tag.
6. Wait for the GitHub release workflow to publish all four platform archives.
7. Calculate the macOS ARM64 and x86_64 archive SHA-256 values.
8. Update the tap formula version, URLs, and checksums; run Homebrew style/audit checks where available; commit and push the tap.

## Files included

The release contains all changes documented in the other 2026-07-18 logs, including character PNGs for Vaughn, Cinder, Zephra, Neris, Brikka, Sergei, and Steven; inventory/stat implementation; new weapon classes; ANSI and Kitty mappings; README changes; `AGENTS.md`; the consistency guide; and dated logs.

## Backtracking

Once published, prefer a new corrective release over rewriting the `v0.3.0` tag. Feature-specific rollback scopes remain documented in the earlier dated logs. The Homebrew formula can be moved to a corrective release by updating both platform URLs/checksums and incrementing the version; do not silently replace published archives under the existing tag.
